Plotting

Parsing

Incoming data is interpreted line-by-line. Each line is split into tokens based on any characters in the "seps" textbox.

If multiple separators appear back-to-back, they are treated as a single separator.

Assuming our separators are " ,:;=", the inputs "A:1 B:2 C:3" and "A 1, B: 2; C=3" are both split into the tokens ["A", "1", "B", "2", "C", "3"]

Key-Value

This plots multiple lines over time. The incoming data is generally formatted in some variation of "KEY1=VALUE KEY2=VALUE KEY3=VALUE".

For example:

GYRO_X:150, HELLO, GYRO_Y:200, GYRO_Z:400, 333

When Tokenized becomes:

["GYRO_X", 150.0, "HELLO", "GYRO_Y", 200.0, GYRO_Z, 400.0, 333.0]

Then interpreted as key-value pairs:

GYRO_X:150.0, GYRO_Y:200.0, GYRO_Z:400.0

Note that the trailing "333" is ignored because it doesn't have a preceding key and "HELLO" is ignored because it doesnt have a value following it.

Key filtering and modifying

If only want to plot certain keys, use the "Keys" textbox. Separate each specific key with any separator in the "Seps" textbox.

For the example above if we set the "Keys" textbox to "GYRO_X,GYRO_Z", the GYRO_Y key would be filtered out and we would be left with:

GYRO_X:150.0, GYRO_Z:400.0

Additionally, the "Keys" textbox can be used to set a scale for a specific key. If we append a "*[SCALE]" to the end of a key, the plotted values will be multiplied by [SCALE]. In our example, if we set the keys to GYRO_X*2,GYRO_Z the result would be:

GYRO_X:300.0, GYRO_Z:400.0
